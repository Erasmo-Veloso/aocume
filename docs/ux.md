# User Experience

## UX Principles

The experience should be:

- Simple
- Fast
- Clear
- Trustworthy
- Mobile-first
- Conversion-focused

Users should always know:

- What AOCUME does.
- Why they should trust the company.
- What action to take next.

# Mobile Experience

Mobile has priority.

Requirements:

- Sticky navigation
- Large buttons
- Touch-friendly spacing
- Optimized images
- Fast loading

---

# Accessibility

Minimum requirements:

- Semantic HTML
- Keyboard navigation
- Color contrast
- Alt text
- Visible focus states
- Responsive typography

---

# UX Rules

Every page should contain at least one primary call-to-action.

Every important section should answer one of these questions:

- What does AOCUME do?
- Why should I trust them?
- How does the process work?
- What can I buy?
- How do I contact them?

If a section does not answer one of these questions, it should not exist.

---

# UX Philosophy

The website is not designed to make users browse endlessly.

It is designed to make them confidently contact AOCUME and begin doing business as quickly as possible.
